# SQL-Based Data Analysis using Filtering, Aggregation & Business Queries

## Objective
This project focuses on analyzing sales data using SQL. The main goal is to apply filtering, aggregation, sorting, and basic business queries to extract meaningful insights from the dataset. It helps in understanding customer behavior, product performance, and overall sales trends.


## Dataset Used
The analysis is performed on the Superstore Sales Dataset available on Kaggle:

https://www.kaggle.com/datasets/vivek468/superstore-dataset-final


## Tools & Technologies
- MySQL Workbench
- Git & GitHub


## Approach Followed

### 1. Data Loading
The dataset was imported into a SQL database for analysis.

### 2. Data Exploration
Basic exploration was done to understand table structure, columns, and sample records.

### 3. Data Filtering
Used WHERE conditions to filter data based on:
- Region
- Category
- Date range
- Sales and profit values

### 4. Data Aggregation
Performed aggregation using:
- SUM for total sales and profit
- COUNT for number of orders
- AVG for average values

### 5. Sorting & Ranking
Identified:
- Top-performing products
- Best-selling categories
- High-value customers

### 6. Business Queries
Solved practical use cases such as:
- Monthly sales trends
- Customer purchase behavior
- Duplicate and data quality checks

### 7. Data Validation
Checked row counts and ensured data consistency across tables.


## Key Insights
- A few product categories contribute the majority of revenue.
- Certain customers generate higher sales compared to others.
- Sales performance varies across different months and regions.
- Aggregation and filtering help in making better business decisions.


## Output
- SQL queries (.sql file)
- Query results screenshots
- Summary report in PDF format


## Learning Outcome
This project helped in strengthening SQL fundamentals and understanding how data analysis is performed in real business scenarios using structured queries.
